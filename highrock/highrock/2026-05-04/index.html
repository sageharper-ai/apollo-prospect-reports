Executive Summary

HighRock's on-page recommendations are anchored around the strategic positioning Monique provided in the SEO research intake: Mid-Atlantic real estate creative agency, with Luxury Real Estate Brand Identity & 3D Visualization, Real Estate Growth Marketing, and Full-Service Creative Partner as the keyword targets. The site's foundation is solid — Drupal Metatag module is configured per-page for Open Graph, all 140+ images have alt text, canonicals are correct site-wide. The gaps are that titles, meta descriptions, H1s, and OG titles currently signal "creative agency" generically rather than the real estate vertical Monique is targeting for SEO growth, and zero pages carry structured data.

Cycle 1 closes both gaps: every title, meta, H1, and OG title for the four priority pages (homepage, /services, /work, /about) is rewritten around the real estate vertical + Mid-Atlantic positioning, and a 5-payload JSON-LD schema bundle adds structured data with audience targeting and Mid-Atlantic service area. None of HighRock's three competitors named in Monique's intake (mojo.biz, warschawski.com, mghus.com) own real estate vertical positioning — that's whitespace HighRock can claim.

Build sequence + cross-page strategy

  • Build Page 1 first (Real Estate Brand Identity) — pairs naturally with HighRock's existing Creative service line and supports the strongest of Monique's keyword targets.
  • Build Page 2 second (3D Visualization) — pairs with Page 1; the keyword target "Brand Identity & 3D Visualization" is dual-page when split this way.
  • Build Page 3 third (Growth Marketing) — covers the campaign-execution side of Monique's keyword phrase 2.
  • Cross-link triangle: all three pages link to each other in their "Related real estate services" footer block. This creates a tight internal linking cluster that signals topical authority on real estate to Google.
  • Update /services parent page — add a "Real Estate Specialty" callout linking to all three sub-pages. This is the bridge between HighRock's broad services landing and the vertical-targeted sub-pages.

Drupal deployment notes for new pages

  • Create as Drupal nodes under the existing service content type (or a new "Service Landing" content type if needed for the H2-section block layout).
  • Each page configures its own Metatag default override (title, meta, OG, Twitter, JSON-LD).
  • Schema.org Metatag submodule (installed per cycle 1) handles per-page Service + FAQ JSON-LD via field tokens.
  • Internal linking: confirm Drupal navigation/menu placement for the three new pages — likely sub-items under /services.

Drupal deployment — Schema.org Metatag submodule

  1. composer require drupal/schema_metatag · drush en schema_metatag
  2. Configuration → Search and metadata → Metatag → enable: schema_organization, schema_professional_service, schema_collection_page, schema_about_page, schema_breadcrumb_list, schema_creative_work
  3. Add defaults: Global scope (Organization) · per-content-type scope (ProfessionalService on homepage + /services; CollectionPage on /work; AboutPage on /about)
  4. Map Drupal field tokens ([node:title], [node:field_*]) for fields that should reflect content edits
  5. Same UI: enable Twitter Card sub-fields on the existing Metatag module — no new install
  6. Validate each URL via Google's Rich Results Test ↗
  7. Submit URLs to Google Search Console for re-indexing
